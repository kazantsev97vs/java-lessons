
    Клиент - Сервер

                                    +------------------+
        +------------------+        |                  |
        |      Клиент      |------->|      Сервер      |-----> ответ
        +------------------+        |                  |
                                    +------------------+

        Сервер работает без остановки - принимает запросы и отдает ответы.

        В качестве сервера будем использовать Apache Tomcat
        - открытая библиотека, которая является web-сервером и еще является контейнером сервлетов

        Сервлет - программа, работающая на сервере,
                  которая взаимодействует с клиентами посредством принципа запрос-ответ.

        Т.е. у нас есть сервер, например, Apache Tomcat - он крутится на сервере без остановки,
        принимает запрос и передает его на сервлет,
        на сервлете мы с помощью JAVA кода с этим запросом что-то делаем и передаем ответ,
        ответ приходит клиенту

        Жизненный цикл сервлета

        Запуск:
            - Tomcat загружает класс сервлета
            - Tomcat создает объект класса сервлета
            - Tomcat вызывает метод init() сервлета

        когда сервер запустится начнется

        Обслуживание запросов:
            - Tomcat получает запрос,
              определяет какому сервлету запрос предназначен,
              передает запрос в метод service()
            - Для каждого запроса отдельного клиента создается свой поток

        когда мы будем останавливать сервер

        Прекращение работы сервлета:
        - Tomcat вызывает метод destroy() на сервлете и сервлет начинает прекращать свое выполнение
