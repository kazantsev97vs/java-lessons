
    Тестирование
        - необходимо для того, чтобы вылавливать bug-и и не допускать их к финальной версии программы.

        bug - не соответсвие ожидаемого результата фактическому.

        Test Case (Тестовый сценарий)
            - Артефакт, описывающий совокупность шагов, конкретных условий и параметров,
              необходимых для проверки реализации тестируемой функции или её части.

        Action -> Expected Result -> Test Result


    Уровни тестирования:

        Модульное (unit) тестирование
            - проверка работы программы на уровне отдельных модулей (классов и методов)
              Процесс проверки корректности работы отдельных частей исходного кода (чаще всего методов)
              программы путем запуска тестов в искуственной среде
              (ОСУЩЕСТВЛЯЕТСЯ РАЗРАБОТЧИКОМ)

        Интеграционное тестирование
            - проверка совестной работы нескольких модулей;

        Системное тестирование
            - проверка работы системы в целом;


    Что делает тест:

        1. Воспроизводит некоторые данные / делает предварительные действия
            (максимально воспроизводит, имитирует реальные условия)

        2. Выполняет тестируемый метод, правильный результат работы которого очевиден автору теста

        3. Выполняет сопоставление полученного результата с ожидаемым (assert)

        Если "ожидание" и "реальность" совпадают, ТЕСТ ПРОЙДЕН.
        Если нет - ТЕСТ ЗАВАЛЕН (чаще всего генерируется специальное исключение - AssertionError)


    Зачем нужно модульное тестирование?

        - убедиться, что разработчик создает методы и классы для конкретных целей;
          Разработка через тестирование (англ. test-driven development, TDD)
              — техника разработки программного обеспечения,
                которая основывается на повторении очень коротких циклов разработки:
                    1) сначала пишется тест, покрывающий желаемое изменение,
                    2) затем пишется код, который позволит пройти тест,
                    3) и под конец проводится рефакторинг нового кода к соответствующим стандартам.

        - снижается число новых ошибок при добавлении новой функциональности

        - тест отражает элементы технического задания
          (некорректное завершение теста сообщает о нарушении технических требований заказчика)
