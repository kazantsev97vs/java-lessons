
    Основные правила:

        1. Каждый тестовый сценарий (test case) - отдельный метод.
           Методы необходимо помечать аннотацией @Test.

        2. Существуют вспомогательные методы с аннотациями:
           @Before,
           @BeforeClass,
           @After,
           @AfterClass

        3. Тестовый класс необходимо называть с суффиксом Test (например, Vector2DTest)

        4. Тестовые методы желательно должны содержать "should" в названии
           (например, sumShouldBePositive)


    Аннотация @Test:

        - объявляет метод тестовым сценарием (обязательно public void).

        Может использовать параметры:

            -- expected - код в тесте проверяется на генерацию определенного исключения

            -- timeout - код в тесте должен работать не более указанного времени (иначе тест завален)


    org.junit.Assert:
        - проверка "ожидание/реальность" (expected/actual)

        Методы (все статические):
            assertTrue()            - проверяет, что метод вернул значение true
            assertFalse()           - проверяет, что метод вернул значение false
            assertEquals()          - проверяет, что фактический результат от метода равен ожидаемому результату
            assertArrayEquals()     - проверяет, тоже самое для массивов
            assertNotEquals()       - проверяет, что фактический результат от метода не совпадает с ожидаемым результатом
            assertSame()
            assertNotSame()

            fail() - при вызове этого метода - тестовый сценарий гарантированно валится
