
    Ignore и Assume.

        Тест, помеченный @Ignore, не выполняется

        Вместо Assert можно использовать Assume (с методами assumeFalse, assumeEquals и др..)

        Assume:
            -- Если проверка верна - тест пройден.
            -- Если проверка неверна - тест игнорируется.


    Эти аннотации используются для того, чтобы помечать методы, как вспомогательные:

    Эти два нужны для того, чтобы подготавливать тестовое окружение:
    @BeforeClass - запускается только один раз при запуске теста (static)
    @Before - запускается перед каждым тестовым методом

    В этих классах, мы можем производить какие-то пост-тестовые задачи,
    например, закрывать какие-то соединения, чистить ресурсы и т.п.:
    @AfterClass - запускается только один раз после того, как отработали все тестовые методы (static)
    @After - запускается после каждого тестового метода

    Жизненный цикл тестирующего класса:

        (При запуске каждого тестового метода, создается объект тестового класса)

        1. @BeforeClass - помечает вспомогательный метод, который запускается только один раз - static,
                          принадлежит не объектам класса, а самому классу.

        2. Для каждого @Test-метода:
              - создание экземпляра тестового класса

        3. @Before - запускается вспомогательный метод с этой аннотацией
                   - будет запускаться для каждого объекта тестового класса

        4. @Test - сам тестовый метод

        5. @After - всомогательный метод - будет запускаться для каждого тестового метода

        6. @AfterClass - после выполнения всех тестовых методов,
                         запускается метод с этой аннотацией,
                         запускается один раз для класса.
